Regreports PIMS — Server Restart Guide

How to start or restart the Flask backend and React frontend  ·  Wastewater Surcharge Compliance System

Overview

Regreports PIMS runs as two separate processes that must both be running for the application to work. The Flask backend serves the API and writes to the database. The React frontend serves the browser interface. Each requires its own terminal window (PowerShell or Command Prompt).

Backend
Python 3.10+ with virtual environment
Directory: C:\Projects\Regreports PIMS\backend
Runs on: http://localhost:5000
Frontend
Node.js 18+ with npm
Directory: C:\Projects\Regreports PIMS\frontend
Runs on: http://localhost:3000
Tip Open two separate PowerShell windows — one for the backend, one for the frontend. Both must stay open while you are using the application. Closing a window stops that process.

1   Starting the Flask Backend

  1. Open a new PowerShell window and navigate to the backend folder. 
    PS C:\>  cd C:\Projects\Regreports PIMS\backend
  2. Activate the Python virtual environment. 
    PS backend>  venv\Scripts\Activate.ps1
    The prompt will change to show (venv) at the start — for example (venv) PS C:\Projects\Regreports PIMS\backend>.
    If activation fails (execution policy error) Run this once in PowerShell as Administrator, then try again:
    Set-ExecutionPolicy RemoteSigned -Scope CurrentUser
  3. Start the Flask server. 
    (venv) PS backend>  python app.py
    You should see output similar to: 
     * Running on http://127.0.0.1:5000
 * Debug mode: on
 * Restarting with stat
Backend is ready when you see Running on http://127.0.0.1:5000 — leave this window open.

Stopping the backend

Press Ctrl + C in the backend PowerShell window. You will see Terminating… and the prompt will return.

Restarting the backend

Stop it with Ctrl + C, then run python app.py again. You do not need to re-activate the virtual environment in the same window — it stays active for the lifetime of that PowerShell session.

Must restart after any Python file change Flask in debug mode auto-reloads most code changes, but if you add a new blueprint, change app.py, or modify models.py, stop and restart manually.

2   Starting the React Frontend

  1. Open a second PowerShell window (keep the backend window open) and navigate to the frontend folder. 
    PS C:\>  cd C:\Projects\Regreports PIMS\frontend
  2. Start the development server. 
    PS frontend>  npm start
    The first start may take 15–30 seconds. When ready, the browser will open automatically at http://localhost:3000.
Frontend is ready when you see Compiled successfully! and the login page appears in your browser.

Stopping the frontend

Press Ctrl + C in the frontend PowerShell window and confirm with Y if prompted.

After code changes

The React dev server watches for file changes and reloads the browser automatically. A manual restart (Ctrl + C then npm start) is rarely needed unless the server itself crashes or a new dependency was installed.

After installing new npm packages If someone adds a new package to package.json, run npm install before npm start.

3   Common Errors & Fixes

Error message Cause Fix
No module named flask Virtual environment not activated Run venv\Scripts\Activate.ps1 first
python: command not found or
python is not recognized		 Running from wrong directory or Python not in PATH Ensure you are in C:\Projects\Regreports PIMS\backend with venv activated
Address already in use (port 5000) A previous backend instance is still running Find and close the other PowerShell window running Flask, or run:
netstat -ano | findstr :5000 then taskkill /PID <id> /F
npm: command not found Running from wrong directory, or Node.js not installed Ensure you are in C:\Projects\Regreports PIMS\frontend
Browser shows blank page or login fails Backend not running or wrong URL Check the backend window — restart if it shows an error. Confirm backend is at localhost:5000
CORS error in browser console Frontend and backend ports do not match proxy config Confirm backend is on port 5000 and frontend proxy in package.json points to http://localhost:5000
Database changes not appearing Backend was not restarted after a model change Stop and restart Flask; SQLAlchemy runs db.create_all() on startup
