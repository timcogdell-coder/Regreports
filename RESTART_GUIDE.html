Regreports PIMS — Developer Guide

Starting the backend & frontend · PostgreSQL access · Quick reference  ·  Linux (ThinkPad X1 Carbon)

Overview

Regreports PIMS runs as two separate processes that must both be running for the app to work. Each requires its own terminal.

Backend
Flask (Python 3.12) + PostgreSQL 16
Directory: /home/tim/Regreports/backend
Runs on: http://localhost:5000
Frontend
React + TypeScript (Node.js)
Directory: /home/tim/Regreports/frontend
Runs on: http://localhost:3000
Tip Open two separate terminal windows — one for the backend, one for the frontend. Both must stay open while using the app. Closing a terminal stops that process.

1   Starting the Flask Backend

  1. Open a terminal and navigate to the backend folder. 
    $  cd /home/tim/Regreports/backend
  2. Activate the Python virtual environment. 
    $  source venv/bin/activate
    The prompt will change to show (venv) at the start.
  3. Start the Flask server. 
    (venv) $  python app.py
    You should see: 
     * Running on http://127.0.0.1:5000
 * Debug mode: on
 * Restarting with stat
Backend is ready when you see Running on http://127.0.0.1:5000 — leave this terminal open.

One-liner (if you prefer)

$  cd /home/tim/Regreports/backend && source venv/bin/activate && python app.py

Stopping / restarting

Press Ctrl + C to stop. Run python app.py again to restart — you don't need to re-activate the venv in the same terminal session. Flask in debug mode auto-reloads most code changes, but restart manually after changing app.py or models.py.

2   Starting the React Frontend

  1. Open a second terminal (keep the backend terminal open) and navigate to the frontend folder. 
    $  cd /home/tim/Regreports/frontend
  2. Start the development server. 
    $  npm start
    The first start may take 15–30 seconds. When ready you'll see Compiled successfully! and the app is available at http://localhost:3000.
Frontend is ready when you see Compiled successfully! — open http://localhost:3000 in the browser.

Stopping

Press Ctrl + C in the frontend terminal.

After installing new packages

Run npm install before npm start if package.json changed.

3   PostgreSQL Access

The database is PostgreSQL 16, running locally on port 5432. Database name: regreports_dev. A tim superuser role exists so you can connect directly from the terminal without a password.

Open an interactive psql session

$  psql -d regreports_dev

Run a one-off query

$  psql -d regreports_dev -c "SELECT * FROM tbl_companies;"

Useful psql commands

CommandWhat it does
\dtList all tables
\d tbl_companiesShow columns for a table
\xToggle expanded (vertical) row display
\qQuit psql
SELECT * FROM tbl_violations WHERE resolved = false;Example query
Table naming All tables use the tbl_ prefix — e.g. tbl_companies, tbl_permits, tbl_samples, tbl_violations.

Backend database URL

The Flask app connects via the DATABASE_URL in /home/tim/Regreports/backend/.env: 

DATABASE_URL=postgresql://postgres:<password>@localhost/regreports_dev

Check PostgreSQL service status

$  pg_lsclusters

The 16 main cluster should show online on port 5432. If it's not running: sudo systemctl start postgresql

5   Common Errors & Fixes

Error Cause Fix
No module named flask Virtual environment not activated Run source venv/bin/activate first
Address already in use (port 5000) A previous backend instance is still running lsof -ti:5000 | xargs kill
Address already in use (port 3000) A previous frontend instance is still running lsof -ti:3000 | xargs kill
Browser shows blank page or login fails Backend not running Check the backend terminal — restart if it shows an error
psql: FATAL: role "tim" does not exist tim PostgreSQL role missing sudo -u postgres createuser --superuser tim
Database changes not appearing Backend not restarted after a model change Stop and restart Flask; db.create_all() runs on startup
