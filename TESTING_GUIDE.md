# Regreports PIMS — Testing Guide

A checklist-based guide for verifying all major features after changes or deployments.
Work through each section in order; later sections depend on data created earlier.

---

## 1. Environment Check

Before testing anything in the browser:

- [ ] Backend is running (`python app.py` — should show `* Running on http://127.0.0.1:5000`)
- [ ] Frontend is running (`npm start` — should open `http://localhost:3000`)
- [ ] PostgreSQL is running and `regreports_dev` database is reachable
- [ ] No red errors in either terminal at startup

## 4. Admin Dashboard

Log in as admin for all tests in this section.

### 4a. Companies Tab

- [ ] Companies tab loads a list of companies on the left
- [ ] Clicking a company row highlights it and shows details on the right panel
- [ ] Detail panel shows: name, contact info, status, associated permits
- [ ] **Add New Company** button (below the list) opens the add form
- [ ] Fill in company name and contact info → Save → company appears in the list
- [ ] Select a company → **Edit** → change a field → Save → change is reflected
- [ ] Select a company that has no permits → **Open Permits** button is disabled (grayed out)
- [ ] Select a company that has permits → **Open Permits** button is enabled → clicking it switches to the Permits tab and shows that company's permits
- [ ] Select a company → **Delete** → a **warning/confirmation dialog** appears → Cancel → company still exists
- [ ] Delete again → Confirm → company is removed from the list

### 4b. Permits Tab

- [ ] Permits tab loads all permits
- [ ] Filter by company works (dropdown or selector)
- [ ] **New Permit** → fill in effective/expiration dates → Save → permit appears in list
- [ ] Open a permit → **Add Limit** → choose a parameter, enter daily max and monthly avg → Save → limit row appears
- [ ] **Batch Add Limits** → add multiple parameters at once → all appear in the table
- [ ] Delete a limit → **confirmation dialog** appears → Confirm → limit removed
- [ ] Expiring permits (within 90 days) visually flagged

### 4c. Samples Tab

- [ ] Samples tab loads all submitted samples
- [ ] Filter by company works
- [ ] Expanding a sample shows its individual analyte results
- [ ] **Delete** a sample result → **confirmation dialog** → Confirm → result removed
- [ ] **Delete** a sample → **confirmation dialog** → Confirm → sample removed
- [ ] **Add Result** to an existing sample → fill in parameter and value → Save → result appears

### 4d. Meters Tab

- [ ] Meters tab shows both flow meters (process and sanitary)
- [ ] Sanitary meter displays unit as **cu.ft.** (not MG or gallons)
- [ ] Process meter displays unit as **MG** (or gallons as configured)
- [ ] **Add Meter** form works → new meter appears
- [ ] **Edit** an existing meter → change a field → Save → change reflected
- [ ] Readings table shows all readings below the meters panel
- [ ] **Add Reading** → enter date, start/end readings, meter assignment, period days → Save → row appears in table
- [ ] **Edit** a reading (inline) → change meter assignment (e.g. process → sanitary) → Save → meter type updates
- [ ] **Delete** a reading → **confirmation dialog** → Confirm → row removed

### 4e. Compliance Tab

- [ ] Compliance summary loads (no blank screen / 500 error)
- [ ] Violations list shows company, parameter, sample date, and severity
- [ ] Filtering by company narrows the list

### 4f. Enforcement Tab

- [ ] Pending enforcement actions list loads
- [ ] **Approve** an action → confirmation or comment field → Submit → action moves out of pending
- [ ] **Override** an action → fill in reason → Submit → override recorded

### 4g. Config Tab

- [ ] POTW config loads (BOD/TSS/Color thresholds, rates)
- [ ] Parameters list loads
- [ ] **Add Parameter** → name and units → Save → appears in list
- [ ] Frequencies list loads

### 4h. Users Tab

- [ ] User list loads
- [ ] **Add User** → fill all fields including role → Save → user appears
- [ ] **Edit** a user → change email → Save → reflected
- [ ] Cannot create duplicate username → error message appears

### 4i. Audit Log Tab

- [ ] Audit log loads with recent entries
- [ ] Entries show timestamp, user, and action description
- [ ] Actions performed in earlier steps (adds, edits, deletes) appear in the log

### 4j. Sampling Schedule Tab

- [ ] Schedule loads showing upcoming due dates per company and parameter
- [ ] Overdue entries visually flagged

### 4k. Notification Bell

- [ ] Bell icon appears in the header
- [ ] Clicking it shows overdue/upcoming sample notifications
- [ ] Clicking a notification navigates to the relevant tab

## 9. Lab Report Parser (CLI)

Requires a Eurofins Columbia PDF lab report.

```bash
cd C:\Projects\lab_report_parsers
pip install -e .

# Text output (human-readable table)
python -m lab_report_parsers path\to\report.pdf

# JSON output
python -m lab_report_parsers path\to\report.pdf --format json

# CSV output
python -m lab_report_parsers path\to\report.pdf --format csv > out.csv
```

- [ ] Text output shows client name, Job ID, sample ID(s), and analyte table
- [ ] `lab` field in JSON output reads **"Regreports PIMS"**
- [ ] Non-detect results (e.g. `<5.0`) show `non_detect: true` and `result: 5.0`
- [ ] CSV has one row per analyte result with all expected columns
- [ ] Running against a file that does not exist prints a clear error and exits with code 1
