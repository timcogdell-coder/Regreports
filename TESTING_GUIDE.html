Regreports PIMS — Testing Guide

Pretreatment Information Management System
Version 1.0 Work through sections in order — later sections depend on data created earlier
1
Environment Check

Verify both servers are running before opening the browser.

  • Backend is running — python app.py shows * Running on http://127.0.0.1:5000
  • Frontend is running — npm start opened http://localhost:3000
  • PostgreSQL is running and regreports_dev database is reachable
  • No red errors in either terminal at startup

4
Admin Dashboard

Log in as admin for all tests in this section.

4a — Companies Tab
  • Company list loads on the left side
  • Clicking a company row highlights it and shows the detail panel on the right
  • Detail panel shows name, contact info, status, and linked permits
  • Add New Company button (below the list) opens the add form
  • Fill in name and contact info → Save → company appears in list
  • Select a company → Edit → change a field → Save → change is reflected
  • Select a company with no permits → Open Permits button is disabled
  • Select a company with permits → Open Permits is enabled → switches to Permits tab showing only that company's permits
  • Delete a company → confirmation dialog appears → Cancel → company still exists
  • Delete again → Confirm → company removed from list
4b — Permits Tab
  • All permits load; filter by company works
  • New Permit → fill in dates → Save → permit appears
  • Open a permit → Add Limit → parameter, daily max, monthly avg → Save → row appears
  • Batch Add Limits → add multiple at once → all appear
  • Delete a limit → confirmation dialog → Confirm → limit removed
  • Permits expiring within 90 days are visually flagged
4c — Samples Tab
  • All submitted samples load; filter by company works
  • Expanding a sample shows its analyte results
  • Add Result to a sample → fill in parameter and value → Save → result appears
  • Delete a sample result → confirmation dialog → Confirm → result removed
  • Delete a sample → confirmation dialog → Confirm → sample removed
4d — Meters Tab
  • Both flow meters (process and sanitary) appear in the meters panel
  • Sanitary meter unit displays as cu.ft. — not MG or gallons
  • Process meter unit displays as MG
  • Add Meter → form saves → new meter appears
  • Edit an existing meter → change a field → Save → change reflected
  • Readings table loads below the meters panel
  • Add Reading → date, start/end, meter, period days → Save → row appears in table
  • Edit a reading (inline) → change meter assignment → Save → meter type updates
  • Delete a reading → confirmation dialog → Confirm → row removed
4e — Compliance Tab
  • Compliance summary loads without errors
  • Violations list shows company, parameter, sample date, and severity
  • Filtering by company narrows the list
4f — Enforcement Tab
  • Pending enforcement actions list loads
  • Approve an action → submit → action moves out of pending
  • Override an action → fill in reason → submit → override recorded
4g — Config Tab
  • POTW config loads (BOD/TSS/Color thresholds and rates)
  • Parameters list loads
  • Add Parameter → name and units → Save → appears in list
  • Frequencies list loads
4h — Users Tab
  • User list loads
  • Add User → all fields including role → Save → user appears
  • Edit a user → change email → Save → reflected
  • Creating a duplicate username shows an error message (not a silent failure)
4i — Audit Log Tab
  • Audit log loads with recent entries
  • Each entry shows timestamp, user, and action description
  • Actions performed in earlier steps appear here
4j — Sampling Schedule Tab
  • Schedule loads with upcoming due dates per company and parameter
  • Overdue entries are visually flagged (red or orange)
4k — Notification Bell
  • Bell icon appears in the header
  • Clicking it shows overdue/upcoming sample notifications
  • Clicking a notification navigates to the relevant section

  • Full sampling schedule table loads with due dates

9
Lab Report Parser (CLI)

Requires a lab report PDF. Run from a terminal.

cd C:\Projects\lab_report_parsers
pip install -e .

# Human-readable table
python -m lab_report_parsers path\to\report.pdf

# JSON output
python -m lab_report_parsers path\to\report.pdf --format json

# CSV — one row per analyte result
python -m lab_report_parsers path\to\report.pdf --format csv > out.csv
  • Text output shows client name, Job ID, sample ID(s), and analyte table
  • lab field in JSON output reads "Regreports PIMS"
  • Non-detect result (e.g. <5.0) shows non_detect: true and result: 5.0
  • CSV has one row per analyte with all expected columns
  • Running against a missing file prints a clear error and exits with code 1
10
Regression Check

These specific issues were fixed during development. Re-verify they have not silently re-broken.

Open Permits button
Select a company that has permits → click Open Permits → Permits tab opens showing only that company's permits. Must not show "please select a company first."
Meter reading type correction
Edit a reading and change its meter assignment (e.g. process → sanitary) → meter type updates correctly in the table after saving.
Sanitary meter unit
Sanitary meter readings display in cu.ft. — not MG, not gallons.
Meters tab layout
Both the meter list panel and the readings table render on the Meters tab. No blank panels or invisible forms.
Company detail panel
Clicking any company in the list always shows the right-side detail panel immediately. No blank screen.
11
Browser Console Check

Open DevTools (F12) → Console after completing the above steps.

  • No red errors from failed API calls (401, 404, 500)
  • No React key warnings for list items
  • No unhandled promise rejections
Tip: If a test fails, check the backend terminal first — Flask prints full stack traces there. The audit log (Admin → Audit Log) confirms whether an action was actually committed to the database. For meter unit issues, verify directly: SELECT id, meter_id, meter_type, unit FROM tbl_flow_meters;
